# Contributing

This is a portfolio-safe demonstration project. Contributions should preserve the public-safety model.

## Development rules

1. Create a feature branch from `main`.
2. Keep changes focused and reviewable.
3. Do not include real personal data, production identifiers, or credentials.
4. Add or update tests for scoring, normalization, and policy behavior.
5. Use mock fixtures for integrations.

## Pull request checklist

- [ ] No secrets or private data.
- [ ] Uses mock data only.
- [ ] Human approval is required before any application submission behavior.
- [ ] Business rules are documented.
- [ ] Tests or test notes are included.
